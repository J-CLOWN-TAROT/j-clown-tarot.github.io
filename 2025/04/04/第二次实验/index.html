Lab2

① 配置静态路由，非常简单

② 通过 ospf 配置默认路由，从而实现局部的自动交换信息

ospf 协议通过把大网络划分成多个 area，从而大大减少了大型网络中路由表中 network 的数量，需要注意的是 每一个 area 之间交流都要经过 area0

//这里的1是ospf的进程号
Router(config)#router ospf 1         
//这里的network ip wildcard area
Router(config-router)#network 192.168.1.0 0.0.0.255 area 0
Router(config-router)#network 192.168.2.0 0.0.0.255 area 0

由于这里的路由器都使用了两个端口，因此都要加入 area0

show ip ospf 查看 OSPF 进程及区域细节的数据
show ip ospf database 查看路由器 OSPF 数据库信息
show ip ospf interface 查看接口 OSPF 信息
show ip ospf neighbor 查看 OSPF 邻居信息
debug ip ospf events 调试 OSPF 事件

encapsulation dot1Q 10

②ACL的机制

R1(config)# access-list 110 permit tcp 192.168.1.0 0.0.0.255 any eq 80
//思科中配置默认是deny any any
R1(config)# access-list 110 deny ip any any

1.ACL从上到下依次匹配

2.ACL默认在 ACL 末尾加上一条deny(permit) ip any any

命令 含义
{no} ip access-group access-list-number out(in) 在出入时引用access-lists中对应number的验证,这里的出入是对于路由器而言
show access-lists / show ip interface 展示access-lists
access-list access-list-number deny(permit) {protocal} source-ip wildcard-mask destination-ip wildcard-mask {eq port} 创建一个access-list-number
no access-list access-list-number 直接删除编号为number整个项，此时端口仍然在这个分组，只是分组内容被删除了
show access-lists access-list-number

Standard IP access list 30
10 permit 192.168.1.0, wildcard bits 0.0.0.255
20 deny any

access-list 30 no 20		 这样可以指定删除某一条规则
ACL类型 编号范围
标准ACL 1-99 和 1300-1999 仅根据源 IP 地址进行过滤
拓展ACL 100-199 和 2000-2699 可以根据源 IP、目标 IP、协议（如 TCP、UDP、ICMP）、以及端口号进行更细致的控制。
服务名称 实际端口 协议
www 80 TCP
https 443 TCP
ftp 21 TCP
telnet 23 TCP
ssh 22 TCP
smpt 25 TCP
domain 53 UDP

Lab4

命令 含义
ip nat inside/outside 进入接口配置模式，启用 NAT，内网接口使用 inside，外部接口使用 outside
静态NAT ip nat inside source static 内部IP 公网IP 把内部IP转成公网IP
动态NAT ip nat pool start-ip end-ip netmask <255.255.255.0> 定义内部 NAT 地址池，pool-name 为地址池的名 字，start-ip和 end-ip为地址池中地址的开始和结束
PAT ip nat inside source list interface <公网接口> overload
show ip nat translations 查看 NAT 转换记录
ip nat inside source list 将访问控制列表access-list-number映射到 NAT 地址池
ip nat inside source list pool 端口映射命令格式
类别 IP 地址范围 子网掩码 CIDR 表示法
A 类 10.0.0.0 – 10.255.255.255 255.0.0.0 10.0.0.0/8
B 类 172.16.0.0 – 172.31.255.255 255.240.0.0 172.16.0.0/12
C 类 192.168.0.0 – 192.168.255.255 255.255.0.0 or 255.255.255.0 192.168.0.0/16 or /24

①三层交换机如果要启用路由功能，要开启ip routing，并且不能进行子端口的划分，而是采用VLAN进行端口的划分
②对于由三层交换机构成的单臂路由，只需要在三层交换机上划分VLAN并且配置IP，交换机只需要划分VLAN，并不需要配置IP
